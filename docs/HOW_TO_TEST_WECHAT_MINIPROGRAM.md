# 如何测试微信小程序前端

> **目标**: 真正测试微信小程序的界面和功能
> **适合人群**: 想要在微信开发者工具中测试小程序的开发者
> **预计时间**: 20-30分钟

---

## 📱 什么是微信小程序测试？

### 对比：后端API测试 vs 小程序测试

| 测试类型 | 测试对象 | 使用工具 | 测试内容 |
|---------|---------|---------|---------|
| **后端API测试** | 服务器接口 | curl命令 | 数据是否正确返回 |
| **小程序测试** | 用户界面 | 微信开发者工具 | 页面显示、用户交互 |

### 举个例子

**后端API测试**（你刚才学的）：
```bash
curl http://localhost:3000/api/products/list
```
→ 返回JSON数据，看起来像：
```json
[{"id":1,"title":"苹果","price":29.90}]
```

**小程序测试**（现在要学的）：
→ 打开小程序，看到漂亮的商品卡片：
```
┌──────────────────┐
│   🍎 新疆阿克苏苹果  │
│   ¥29.90         │
│   [加入购物车]    │
└──────────────────┘
```

## 第一步：准备测试环境

### 1.1 确认后端服务器正在运行

**打开第一个命令行窗口**：
```bash
cd E:/AI/cc+glm/backend
npm start
```

等待看到：
```
🚀 服务器启动成功
📍 端口: 3000
```

> ⚠️ **重要**：小程序必须依赖后端服务器，先启动后端！

### 1.2 打开微信开发者工具

如果你还没安装：
1. 访问：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载并安装微信开发者工具

## 第二步：导入小程序项目

### 2.1 打开微信开发者工具

启动后的界面：
```
┌──────────────────────────────────┐
│  微信开发者工具                    │
│                                  │
│  [+]  [导入]  [打开]              │
│                                  │
│  最近使用：                        │
│  - 项目1                          │
│  - 项目2                          │
└──────────────────────────────────┘
```

### 2.2 导入项目

**1. 点击"+"号或"导入"**

**2. 填写项目信息**：
- **项目名称**：在线购物测试
- **目录**：选择 `E:/AI/cc+glm/Wechat_Online_Shopping`
- **AppID**：选择"测试号"或填入你的AppID

**3. 点击"导入"**

### 2.3 配置开发环境

**非常重要！**

**1. 点击右上角"详情"按钮**

**2. 选择"本地设置"标签**

**3. 勾选以下选项**：
- ✅ 不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书

> 💡 **为什么要勾选？**
> - 因为你的后端在 `localhost:3000`（本地）
> - 正式上线需要用HTTPS和备案域名
> - 开发测试阶段可以勾选这个选项绕过检查

## 第八步：使用调试工具

### 8.1 打开调试器

点击工具栏的"调试器"按钮

调试器包含多个标签：

### 8.2 Console（控制台）

**作用**：查看日志输出

**常见日志**：
```javascript
// API请求
[请求] GET /api/products/list

// 成功响应
[响应] {"code":"Success", "data":[...]}

// 错误信息
[错误] request:fail
```

### 8.3 Network（网络）

**作用**：查看所有API请求

**如何使用**：
1. 点击"Network"标签
2. 在小程序中操作（如切换页面）
3. 查看Network中出现的请求

**示例**：
```
GET /api/products/list
Status: 200
Time: 150ms
Response: {"code":"Success", "data":[...]}
```

### 8.4 Storage（存储）

**作用**：查看本地存储的数据

**常见存储**：
- `token`: 用户登录令牌
- `userInfo`: 用户信息
- `cart`: 购物车数据

### 8.5 AppData（应用数据）

**作用**：查看页面的数据状态

**如何使用**：
1. 点击"AppData"标签
2. 展开Pages
3. 选择当前页面
4. 查看data中的数据

**示例**：
```javascript
Pages: {
  index: {
    data: {
      productList: [...],
      currentTab: 0
    }
  }
}
```

## 第九步：常见问题排查

### 问题1：小程序无法连接后端

**症状**：
- 商品列表不显示
- 提示"网络错误"
- Console显示 `request:fail`

**解决方法**：
1. 确认后端服务器正在运行
   ```bash
   curl http://localhost:3000/health
   ```
2. 确认已勾选"不校验合法域名"
3. 查看小程序配置的API地址
   - 文件：`Wechat_Online_Shopping/config/index.js`
   - 应该是：`apiBaseURL: 'http://localhost:3000/api'`

### 问题2：登录失败

**症状**：
- 提示"账号或密码错误"
- 点击登录没反应

**解决方法**：
1. 确认账号密码正确：
   - 用户名：testuser
   - 密码：123456
2. 查看Console的错误信息
3. 测试后端登录API：
   ```bash
   curl -X POST http://localhost:3000/api/users/login \
     -H "Content-Type: application/json" \
     -d '{"username":"testuser","password":"123456"}"
   ```

### 问题3：图片不显示

**症状**：
- 商品图片位置空白
- 显示"图片加载失败"

**原因**：
- 测试数据的图片URL是假的（`https://example.com/...`）

**解决方法**：
- 这是正常的，不影响功能测试
- 如果要显示真实图片，需要上传图片到服务器

### 问题4：页面白屏

**症状**：
- 模拟器显示白色空白页面
- Console有报错

**解决方法**：
1. 查看Console的错误信息
2. 常见原因：
   - JavaScript代码错误
   - 页面路径配置错误
3. 尝试重新编译：
   - 点击"编译" → "清缓存" → "清除全部缓存"
   - 再次点击"编译"
