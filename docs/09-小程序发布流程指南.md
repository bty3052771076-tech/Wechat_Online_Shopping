### 1.4 服务器域名配置

#### 配置服务器域名

1. 登录小程序后台
2. 进入「**开发**」→「**开发管理**」→「**开发设置**」
3. 找到「**服务器域名**」
4. 点击「**修改**」
5. 配置以下域名：

```
request合法域名：
https://api.yourdomain.com
https://your-api-server.com

socket合法域名：
wss://api.yourdomain.com

uploadFile合法域名：
https://upload.yourdomain.com

downloadFile合法域名：
https://static.yourdomain.com
```

**域名要求：**
- ✅ 必须是HTTPS协议
- ✅ 必须已备案
- ✅ SSL证书有效
- ✅ 域名解析正确

#### 业务域名配置

```
业务域名：
www.yourdomain.com
yourdomain.com
```

**用途：** 小程序内打开网页

## 2. 发布流程

### 2.1 完整发布流程图

```
开发完成
    ↓
本地测试
    ↓
真机测试
    ↓
上传代码（开发者工具）
    ↓
测试版体验
    ↓
提交审核（小程序后台）
    ↓
等待审核（1-7个工作日）
    ↓
审核通过
    ↓
发布上线
    ↓
所有用户可访问
```

### 2.2 步骤详解

#### 步骤1：上传代码

**在微信开发者工具中：**

1. **检查代码**
   - 运行一遍，确保无报错
   - 测试主要功能
   - 检查控制台有无错误

2. **点击「上传」**
   - 工具栏 → 「上传」按钮
   - 或快捷键：Ctrl/Cmd + Shift + U

3. **填写版本信息**

```
版本号：1.0.0
项目备注：初始版本，包含以下功能：
- 用户登录和注册
- 商品浏览和搜索
- 购物车管理
- 订单创建和管理
- 个人中心
```

**版本号规范：**
- 主版本号.次版本号.修订号
- 例如：1.0.0（第一个正式版本）

4. **点击「上传」**
   - 等待上传完成
   - 显示「上传成功」

#### 步骤2：提交审核

**在微信小程序后台：**

1. **登录后台**
   - 网址：https://mp.weixin.qq.com/
   - 使用小程序账号登录

2. **进入版本管理**
   - 左侧菜单 → 「**管理**」→「**版本管理**」

3. **查看开发版本**
   - 找到刚才上传的版本
   - 状态显示：「**开发中**」
   - 点击「**提交审核**」

4. **填写审核信息**

```
┌──────────────────────────────────────┐
│  提交审核                              │
│  ┌────────────────────────────────┐  │
│  │ 测试账号（选填）：               │  │
│  │ 账号：test001                   │  │
│  │ 密码：123456                    │  │
│  │                                │  │
│  │ 测试截图（必需）：               │  │
│  │ [上传首页截图]                  │  │
│  │ [上传功能页面截图]              │  │
│  │                                │  │
│  │ 功能页面（必需）：               │  │
│  │ 首页：pages/home/home           │  │
│  │ 分类页：pages/category/index    │  │
│  │ 购物车：pages/cart/index        │  │
│  │ 个人中心：pages/usercenter/index│  │
│  └────────────────────────────────┘  │
│                                        │
│  [提交审核]  [保存草稿]                 │
└──────────────────────────────────────┘
```

**测试账号说明：**
- 如果小程序需要登录，必须提供测试账号
- 方便审核人员测试功能
- 不要使用个人账号

**测试截图要求：**
- 截图清晰，展示主要功能
- 至少3张截图
- 格式：jpg、png
- 大小：不超过2MB

**功能页面：**
- 填写主要的页面路径
- 至少3个核心页面
- 确保路径正确

5. **勾选协议**
   - 阅读《小程序审核指南》
   - 勾选「**我已阅读……**」
   - 点击「**提交审核**」

#### 步骤3：等待审核

**审核时间：**
- **正常审核**：1-7个工作日
- **加急审核**：特定情况可申请（极少）

**审核状态查询：**

1. 进入「**版本管理**」
2. 查看「**审核版本**」
3. 状态显示：
   - 「审核中」：正在审核
   - 「审核通过」：可以发布
   - 「审核驳回」：需要修改后重新提交

**审核通知：**
- 短信通知（注册手机号）
- 邮件通知（注册邮箱）
- 站内消息（小程序后台）

#### 步骤4：审核通过后发布

**审核通过后：**

1. **查看审核结果**
   - 登录小程序后台
   - 进入「**版本管理**」
   - 状态显示：「**审核通过**」

2. **点击「发布」**
   - 找到审核通过的版本
   - 点击「**发布**」按钮

3. **确认发布**
   - 阅读发布须知
   - 点击「**确认发布**」

4. **发布完成**
   - 小程序正式上线
   - 所有用户可搜索访问

## 3. 审核要求

### 3.1 基本要求

#### 资质要求

| 账号类型 | 必需资质 |
|---------|---------|
| **个人** | 身份证 |
| **企业** | 营业执照、组织机构代码证 |
| **政府** | 组织机构代码证 |
| **媒体** | 媒体许可证 |
| **其他组织** | 组织机构代码证 |

#### 内容要求

**✅ 允许的内容：**
- 正常的商业服务
- 便民工具
- 信息展示
- 生活服务

**❌ 禁止的内容：**
- 违法违规内容
- 虚假宣传
- 欺诈信息
- 色情暴力
- 政治敏感
- 侵犯版权
- 虚假红包活动

### 3.2 功能要求

#### 必须有的功能

- [ ] **首页**：展示小程序主要功能
- [ ] **页面导航**：清晰的导航结构
- [ ] **用户反馈**：提供反馈渠道
- [ ] **服务说明**：说明服务内容

#### 个人账号限制

**个人账号不能有的功能：**
- ❌ 电商交易（但可以展示商品）
- ❌ 付费服务
- ❌ 会员系统
- ❌ 获取用户手机号

**个人账号可以有的功能：**
- ✅ 信息展示
- ✅ 内容浏览
- ✅ 工具类功能
- ✅ 个人笔记

### 3.3 UI/UX要求

#### 界面设计

**✅ 好的设计：**
- 界面简洁清晰
- 导航逻辑合理
- 操作流畅自然
- 文字清晰可读
- 图片清晰不模糊

**❌ 差的设计：**
- 界面混乱无章
- 按钮位置不合理
- 文字过小或颜色对比差
- 图片模糊或加载失败

#### 用户体验

**基本要求：**
- 页面加载速度快（<3秒）
- 操作响应及时（<1秒）
- 无明显卡顿
- 无闪退问题

### 3.4 技术要求

#### 性能要求

- **启动时间**：<3秒
- **首屏加载**：<2秒
- **页面切换**：<1秒
- **内存占用**：<150MB
- **包大小**：
  - 主包：<2MB
  - 单个分包：<2MB
  - 总包大小：<20MB

#### 兼容性要求

- 支持iOS 10.0+
- 支持Android 5.0+
- 适配不同屏幕尺寸
- 适配有刘海屏的设备

## 4. 常见驳回原因

### 4.1 内容违规

#### 1. 虚假宣传

**驳回原因：**
```
你的小程序涉及虚假宣传，商品描述与实际不符。
```

**解决方法：**
- 确保所有描述真实准确
- 不要使用夸大宣传
- 提供真实的商品信息

#### 2. 侵权内容

**驳回原因：**
```
你的小程序涉嫌侵犯他人版权/商标权。
```

**解决方法：**
- 使用正版素材
- 不使用他人商标
- 注明内容来源

#### 3. 违法违规

**驳回原因：**
```
你的小程序包含违法违规内容。
```

**解决方法：**
- 删除违规内容
- 重新提交审核
- 严重者会被封禁

### 4.2 功能问题

#### 1. 功能不完整

**驳回原因：**
```
你的小程序功能不完整，无法正常使用。
```

**解决方法：**
- 完善所有功能
- 确保流程完整
- 进行充分测试

#### 2. 页面空白

**驳回原因：**
```
你的小程序存在页面空白问题。
```

**解决方法：**
- 检查所有页面
- 修复空白页面
- 确保内容正常显示

#### 3. 无法使用

**驳回原因：**
```
你的小程序无法正常使用。
```

**解决方法：**
- 检查API调用
- 修复报错
- 确保功能正常

### 4.3 资质问题

#### 1. 资质不全

**驳回原因：**
```
你的小程序缺少必要的资质文件。
```

**解决方法：**
- 补充所需资质
- 更新营业执照
- 上传许可证

#### 2. 类目不符

**驳回原因：**
```
你的小程序服务类目与实际功能不符。
```

**解决方法：**
- 选择正确的服务类目
- 确保类目与功能一致
- 提供相应资质

### 4.4 技术问题

#### 1. 域名未备案

**驳回原因：**
```
你的服务器域名未备案。
```

**解决方法：**
- 完成域名ICP备案
- 备案通过后重新提交
- 或使用已备案域名

#### 2. HTTPS证书无效

**驳回原因：**
```
你的服务器SSL证书无效。
```

**解决方法：**
- 更新SSL证书
- 确保证书有效
- 使用正确的证书链

#### 3. 接口调用失败

**驳回原因：**
```
你的小程序接口调用失败。
```

**解决方法：**
- 检查服务器状态
- 修复接口错误
- 确保数据正常返回

### 4.5 如何避免被驳回

#### 发布前检查清单

## 5. 发布后运营

### 5.1 数据分析

#### 查看数据

**小程序后台 → 统计**

1. **用户分析**
   - 新增用户
   - 活跃用户
   - 留存率
   - 用户来源

2. **页面分析**
   - 页面访问量
   - 页面停留时间
   - 页面跳出率

3. **功能分析**
   - 功能使用次数
   - 功能转化率
   - 功能错误率

#### 数据指标

**核心指标：**
- **DAU（日活跃用户）**：每天使用小程序的用户数
- **MAU（月活跃用户）**：每月使用小程序的用户数
- **留存率**：用户再次使用的比例
- **转化率**：完成目标行为的比例

### 5.2 用户反馈

#### 收集反馈

**途径：**

1. **客服消息**
   - 小程序内客服功能
   - 及时回复用户问题

2. **意见反馈**
   - 添加反馈入口
   - 收集用户建议

3. **评价系统**
   - 用户评分和评论
   - 分析评价数据

4. **数据监控**
   - 监控异常数据
   - 及时发现问题

#### 处理反馈

**流程：**

```
收集反馈 → 分类整理 → 优先级排序 → 处理问题 → 回复用户
```

### 5.3 版本迭代

#### 迭代周期

**建议：**
- **初期**：1-2周一个小版本
- **稳定期**：1个月一个大版本
- **重大更新**：根据需求确定

#### 版本号管理

```
1.0.0 → 1.0.1 → 1.1.0 → 1.2.0 → 2.0.0
  bug修复  新功能  新功能  重大更新
```

## 6. 版本更新

### 6.1 更新流程

```
开发新版本
    ↓
本地测试
    ↓
上传代码（新版本号）
    ↓
提交审核
    ↓
审核通过
    ↓
发布新版本
```

### 6.2 版本号规则

**语义化版本：**

```
主版本号.次版本号.修订号

例如：1.2.3

- 主版本号：不兼容的API修改
- 次版本号：向下兼容的功能性新增
- 修订号：向下兼容的问题修正
```

**示例：**

```
1.0.0 → 第一个正式版本
1.0.1 → 修复bug
1.1.0 → 新增功能
2.0.0 → 重大更新
```

### 6.3 灰度发布

**什么是灰度发布？**

逐步放出新版本，先让少量用户使用，确认无问题后全量发布。

**设置步骤：**

1. 上传新版本
2. 提交审核
3. 审核通过后，选择「**分阶段发布**」
4. 设置灰度比例：
   - 10% → 30% → 50% → 100%
5. 观察数据，逐步扩大范围

**优势：**
- 降低风险
- 及时发现问题
- 平滑升级

## 7. 注意事项

### 7.1 审核次数限制

**规则：**
- 每个月有**多次审核机会**（具体数量未公开）
- 驳回次数过多会被限制
- 限制期无法提交审核

**建议：**
- 发布前充分测试
- 确保无问题再提交
- 避免频繁提交

### 7.2 审核时间

**时间要求：**
- 工作日提交审核较快
- 节假日提交可能延迟
- 高峰期可能延长

**建议：**
- 提前规划发布时间
- 避开节假日提交
- 留出充足的审核时间

### 7.3 版本回退

**回退规则：**
- 每次回退占用审核次数
- 审核失败次数过多会限制
- 回退后重新审核需要时间

**建议：**
- 谨慎回退
- 确保新版本稳定后再发布
- 保留回退余地

### 7.4 数据安全

**注意：**
- 体验版和正式版数据共享
- 发布前确保数据迁移正确
- 备份重要数据

### 7.5 合规运营

**遵守规则：**
- 不要诱导用户分享
- 不要强制关注
- 不要虚假宣传
- 不要违规收集信息

## 8. 总结

### 发布流程回顾

```
1. 开发完成 → 本地测试
2. 真机测试 → 修复bug
3. 上传代码 → 填写版本信息
4. 提交审核 → 等待审核结果
5. 审核通过 → 发布上线
6. 数据监控 → 持续优化
```

### 关键要点

1. ✅ 发布前充分测试
2. ✅ 确保符合审核要求
3. ✅ 填写正确的审核信息
4. ✅ 耐心等待审核结果
5. ✅ 发布后持续优化

### 常见问题速查

| 问题 | 解决方法 |
|------|---------|
| 审核被驳回 | 根据驳回原因修改后重新提交 |
| 审核时间过长 | 等待1-7个工作日，或联系客服 |
| 上传失败 | 检查网络，清除缓存重试 |
| 域名未备案 | 完成ICP备案后再提交 |
| 功能无法使用 | 修复bug后重新上传 |
