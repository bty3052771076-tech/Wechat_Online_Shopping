## 📋 项目概述

本项目是一个完整的微信小程序在线购物系统，包含消费者端和管理端两大模块，涵盖商品浏览、购物车、订单管理、支付、物流、售后等完整的电商功能链路。

### 核心功能

#### 消费者端功能
- ✅ 用户登录与注册（手机号/邮箱 + 微信授权）
- ✅ 商品浏览与搜索（分类、推荐、搜索）
- ✅ 购物车管理（增删改查、数量修改、全选）
- ✅ 订单结算与支付（微信支付）
- ✅ 订单管理与售后（退款/退货申请）
- ✅ 个人中心（订单、地址、优惠券等）

#### 管理端功能
- ✅ 商品管理（CRUD、上下架、库存管理）
- ✅ 订单管理（查询、发货、售后处理）
- ✅ 用户管理（查看用户信息、备注）
- ✅ 配送管理（区域配送费设置）

### 技术栈

| 类别 | 技术选型 |
|------|---------|
| **前端** | 微信小程序原生框架 + TDesign 组件库 |
| **后端** | Node.js (NestJS/Express) 或 微信云开发 |
| **数据库** | MySQL + Redis |
| **存储** | 腾讯云 COS / 阿里云 OSS |
| **支付** | 微信支付 |
| **物流** | 快递100 API |

## 📌 注意事项

### 1. 账号注册注意事项
- **微信小程序**: 必须使用企业账号才能开通支付功能
- **微信支付**: 需要完成企业认证后才能申请
- **域名备案**: 服务器在大陆必须完成ICP备案，需要20个工作日
- **云服务**: 建议提前注册并完成实名认证

### 2. 开发注意事项
- 小程序分包限制: 主包<2MB，单个分包<2MB，总包<20MB
- 接口请求域名必须在小程序后台配置
- 支付功能必须在后台配置支付目录
- 用户隐私协议必须符合最新规范

### 3. 部署注意事项
- 生产环境必须使用HTTPS
- SSL证书必须由受信任的CA机构签发
- 数据库密码必须足够复杂
- 敏感信息（密钥、密码）不要提交到代码仓库

### 4. 安全注意事项
- 用户密码必须加密存储（bcrypt）
- 敏感接口必须进行权限验证
- 支付回调必须验证签名
- 防止SQL注入、XSS攻击
- 实施接口限流防止恶意请求
