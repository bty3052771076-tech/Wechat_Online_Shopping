## 1. 开发工具概述

### 1.1 什么是微信开发者工具？

**微信开发者工具**是微信官方提供的**集成开发环境（IDE）**，专门用于开发、调试、测试和发布微信小程序。

### 1.2 为什么要使用微信开发者工具？

| 原因 | 说明 |
|------|------|
| **唯一性** | 微信只认可通过此工具开发的代码 |
| **必需性** | 上传代码、提交审核必须使用此工具 |
| **完整性** | 集成开发、调试、预览、上传于一体 |
| **官方性** | 微信官方维护，与小程序API同步更新 |
| **免费性** | 完全免费，无任何限制 |

### 1.3 工具版本

| 版本类型 | 说明 | 适用场景 |
|---------|------|---------|
| **稳定版** | 经过充分测试 | 日常开发（推荐） |
| **预发布版** | 新功能测试 | 尝试新特性 |
| **开发版** | 最新功能 | 极客体验 |

**建议：日常开发使用稳定版！**

## 2. 安装与启动

### 2.1 下载安装

#### Windows 系统

1. **访问下载页面**
   - 网址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 选择「**Windows**」版本
   - 下载 `wechat_devtools_x.xx.exe`

2. **运行安装程序**
   - 双击安装文件
   - 选择安装路径（建议：`C:\Program Files (x86)\Tencent\微信web开发者工具`）
   - 点击「**安装**」
   - 等待安装完成

#### macOS 系统

1. **访问下载页面**
   - 网址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 选择「**macOS**」版本
   - 下载 `wechat_devtools_x.xx.dmg`

2. **安装应用**
   - 打开 `.dmg` 文件
   - 将「微信开发者工具」拖到「应用程序」文件夹
   - 在启动台打开应用

### 2.2 首次启动

#### 启动工具

1. **Windows**: 双击桌面图标或从开始菜单启动
2. **macOS**: 从启动台或应用程序文件夹启动

#### 微信扫码登录

⚠️ **重要：必须使用微信扫码登录！**

1. 工具启动后会显示二维码
2. 使用微信扫描二维码
3. 在手机上确认登录
4. 登录成功

**为什么要扫码登录？**
- 验证开发者身份
- 关联你的微信账号
- 同步小程序权限

## 5. 开发调试

### 5.1 代码编辑

#### 编辑代码

微信开发者工具内置代码编辑器：

**优点：**
- 与小程序API完美集成
- 实时预览效果
- 自动补全
- 语法高亮

**缺点：**
- 功能不如VS Code强大
- 插件生态较弱

**建议：**
- **简单项目**：直接用开发者工具
- **复杂项目**：VS Code + 开发者工具（配合使用）

#### 文件类型支持

| 文件类型 | 说明 | 编辑支持 |
|---------|------|---------|
| `.js` | JavaScript逻辑 | ✅ 完整支持 |
| `.json` | 配置文件 | ✅ 完整支持 |
| `.wxml` | 页面结构 | ✅ 专用支持 |
| `.wxss` | 页面样式 | ✅ 专用支持 |
| `.wxs` | 页面脚本 | ✅ 专用支持 |

### 5.2 实时预览

#### 自动编译

```
修改代码 → 自动保存 → 自动编译 → 自动刷新模拟器
```

**配置自动编译：**
```
设置 → 编辑设置 →
☑ 修改文件时自动保存
☑ 修改文件时自动编译
```

#### 手动编译

如果自动编译失效，可以手动触发：

1. 点击「**编译**」按钮
2. 或使用快捷键：Ctrl/Cmd + B

### 5.3 调试技巧

#### Console调试

**打印日志：**
```javascript
console.log('普通日志');
console.info('信息日志');
console.warn('警告日志');
console.error('错误日志');
```

**查看日志：**
1. 打开调试器
2. 切换到「**Console**」面板
3. 查看输出的日志

#### 断点调试

**设置断点：**
1. 打开调试器「**Sources**」面板
2. 找到对应的 `.js` 文件
3. 点击行号设置断点

**调试操作：**
- F8: 继续
- F10: 单步跳过
- F11: 单步进入
- Shift + F11: 单步退出

#### Wxml面板

**元素审查：**
1. 打开调试器「**Wxml**」面板
2. 选择元素查看
3. 查看元素属性、样式、数据绑定

#### Storage面板

**查看本地存储：**
```javascript
// 存储数据
wx.setStorageSync('key', 'value');

// 读取数据
const value = wx.getStorageSync('key');

// 删除数据
wx.removeStorageSync('key');
```

在「**Storage**」面板可以查看所有存储的数据。

## 6. 预览测试

### 6.1 预览功能

#### 预览到手机

**步骤：**

1. 点击工具栏「**预览**」按钮
2. 生成预览二维码

**用途：**

- 分享给他人体验
- 收集反馈

### 6.2 体验版

#### 什么是体验版？

**体验版**是小程序的测试版本，可以分享给其他人体验。

#### 生成体验版

**步骤：**

1. 点击「**上传**」按钮
2. 填写版本号和备注
3. 点击「**上传**」
4. 等待上传完成
5. 在微信小程序后台
6. 进入「**版本管理**」
7. 将开发版设为体验版
8. 生成体验版二维码

**体验版 vs 开发版：**

| 类型 | 获取方式 | 有效期 |
|------|---------|--------|
| **开发版** | 工具「预览」扫码 | 扫码后即时可用 |
| **体验版** | 后台生成二维码 | 长期有效（可随时关闭） |

## 8. 版本管理

### 8.1 小程序版本类型

```
开发版本 → 体验版本 → 审核版本 → 线上版本
```

#### 开发版本

- **来源**：开发者工具上传
- **状态**：开发中
- **可见性**：仅开发者可见
- **用途**：开发测试

#### 体验版本

- **来源**：从开发版本选定
- **状态**：测试中
- **可见性**：通过体验版二维码访问
- **用途**：内部测试、收集反馈

#### 审核版本

- **来源**：提交审核
- **状态**：审核中
- **时间**：1-7个工作日
- **结果**：通过 / 驳回

#### 线上版本

- **来源**：审核通过后发布
- **状态**：已发布
- **可见性**：所有用户
- **用途**：正式运营

### 8.2 版本回退

**已发布的版本可以回退：**

1. 登录小程序后台
2. 进入「**版本管理**」
3. 找到「**线上版本**」
4. 点击「**回退**」
5. 选择要回退的版本

⚠️ **注意：**
- 每次回退会占用审核次数
- 审核失败次数过多会被限制

### 8.3 灰度发布

**什么是灰度发布？**

灰度发布是指让部分用户先使用新版本，逐步扩大到全部用户。

**设置灰度：**

1. 登录小程序后台
2. 进入「**版本管理**」
3. 点击「**分阶段发布**」
4. 设置灰度比例（10%、30%、50%、100%）

## 10. 常见问题

### Q1: 预览时提示"不是合法域名"？

**原因：** 开发阶段未配置服务器域名白名单

**解决方法：**

1. 点击「**详情**」
2. 找到「**本地设置**」
3. 勾选「**不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**

⚠️ **注意：** 上线前必须配置合法域名！

### Q2: 模拟器显示空白？

**可能原因：**

1. AppID配置错误
2. 基础库版本不兼容
3. 代码有语法错误

**解决方法：**

1. 检查 `project.config.json` 中的 `appid`
2. 查看调试器的Console是否有错误
3. 尝试降低基础库版本

### Q3: 代码修改后没有反应？

**解决方法：**

1. 点击「**清缓存**」→「**清除文件缓存**」
2. 点击「**编译**」按钮
3. 重启开发者工具

### Q4: 上传失败？

**可能原因：**

1. 网络问题
2. 项目太大
3. 权限问题

**解决方法：**

1. 检查网络连接
2. 清除缓存后重试
3. 检查是否有管理员权限

### Q5: 找不到某个API？

**可能原因：**

1. 基础库版本太低
2. API名称拼写错误

**解决方法：**

1. 升级基础库版本
2. 查阅官方文档确认API名称
3. 使用 `wx.canIUse()` 检查API可用性

### Q6: 如何查看小程序API文档？

**方法一：**

在开发者工具中：
1. 点击「**帮助**」→「**开发者文档**」
2. 在浏览器中打开文档

**方法二：**

直接访问：https://developers.weixin.qq.com/miniprogram/dev/api/

### Q7: 体验版和正式版的区别？

| 特性 | 体验版 | 正式版 |
|------|--------|--------|
| **访问方式** | 扫码访问 | 搜索/扫码访问 |
| **可见性** | 仅扫码者可见 | 所有用户可见 |
| **用途** | 内部测试 | 正式运营 |
| **数据** | 共享测试数据 | 独立生产数据 |

## 11. 最佳实践

### 11.1 开发工作流

```
1. 本地开发
   ↓
2. 工具中调试
   ↓
3. 预览到手机测试
   ↓
4. 上传代码
   ↓
5. 生成体验版
   ↓
6. 内部测试
   ↓
7. 提交审核
   ↓
8. 审核通过
   ↓
9. 发布上线
```

### 11.2 版本管理建议

**版本号规则：**

```
主版本号.次版本号.修订号

例如：1.2.3
- 主版本号：重大功能更新（1 → 2）
- 次版本号：新功能添加（1.0 → 1.1）
- 修订号：bug修复（1.0.0 → 1.0.1）
```

**建议：**
- 每次发布前增加版本号
- 在版本备注中说明更新内容
- 重要更新记录在更新日志中

### 11.3 性能优化

**代码体积优化：**

1. 启用「**压缩代码**」
2. 使用分包加载
3. 删除无用代码
4. 压缩图片资源

**运行性能优化：**

1. 减少 setData 调用
2. 使用虚拟列表
3. 图片懒加载
4. 合理使用缓存
