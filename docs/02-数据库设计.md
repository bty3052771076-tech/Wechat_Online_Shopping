## 1. 数据库选型说明

### 1.1 关系型数据库选择：MySQL

**选择理由：**

1. **成熟稳定**: MySQL 是业界最广泛使用的开源关系型数据库，具有强大的社区支持和丰富的文档资源
2. **高性能**: 优化了读取性能，适合电商系统大量查询的场景
3. **事务支持**: 完整的 ACID 特性支持，确保订单、支付等核心业务的可靠性
4. **成本效益**: 开源免费，降低项目成本
5. **生态完善**: 与各种开发语言、框架、监控工具都有良好的集成
6. **数据一致性**: 强一致性保证，适合对数据准确性要求高的电商场景

### 1.2 缓存数据库选择：Redis

**选择理由：**

1. **高性能**: 基于内存存储，读写速度极快（QPS 可达 10万+）
2. **丰富数据结构**: 支持 String、Hash、List、Set、Sorted Set 等多种数据结构
3. **缓存热点数据**: 商品信息、分类信息等高频访问数据
4. **分布式锁**: 解决订单并发问题，防止超卖
5. **会话管理**: 存储用户登录状态和购物车数据
6. **计数器**: 商品浏览量、销量等实时统计
7. **发布订阅**: 实现订单状态变更通知

### 1.3 数据分片策略

**当前阶段（单库单表）：**
- 用户规模：< 10万
- 商品规模：< 10万
- 日订单量：< 1万

**未来扩展策略（水平分片）：**
- **用户表**: 按 user_id 取模分片（user_id % 16）
- **订单表**: 按用户ID分片，保证同一用户订单在同一分片
- **商品表**: 按商品分类分片，便于分类查询优化
- **购物车表**: 按用户ID分片

| city | VARCHAR | 50 | 否 | NULL | 城市 |
| language | VARCHAR | 20 | 否 | NULL | 语言 |

## 3. Redis 缓存设计

### 3.3 缓存更新策略

#### 3.3.1 Cache-Aside Pattern（旁路缓存）

**读取流程**:
1. 先查询缓存
2. 缓存命中，直接返回
3. 缓存未命中，查询数据库
4. 将数据写入缓存
5. 返回数据

**更新流程**:
1. 先更新数据库
2. 删除缓存（而非更新缓存）

**示例代码**:
```python
def get_product(product_id):
    # 1. 查询缓存
    cache_key = f"mall:product:info:{product_id}"
    product = redis.hgetall(cache_key)

    if product:
        return product

    # 2. 查询数据库
    product = db.query("SELECT * FROM products WHERE id = %s", product_id)

    # 3. 写入缓存
    redis.hmset(cache_key, product)
    redis.expire(cache_key, 3600)  # 1小时过期

    return product

def update_product(product_id, data):
    # 1. 更新数据库
    db.execute("UPDATE products SET ... WHERE id = %s", product_id)

    # 2. 删除缓存
    cache_key = f"mall:product:info:{product_id}"
    redis.delete(cache_key)
```

#### 3.3.2 缓存穿透处理

**解决方案**: 布隆过滤器 + 空值缓存

```python
def get_product_with_bloom_filter(product_id):
    # 1. 布隆过滤器判断是否存在
    if not bloom_filter.exists(product_id):
        return None

    # 2. 查询缓存
    cache_key = f"mall:product:info:{product_id}"
    product = redis.hgetall(cache_key)

    # 3. 缓存空值处理
    if product == "NULL":
        return None

    if product:
        return product

    # 4. 查询数据库
    product = db.query("SELECT * FROM products WHERE id = %s", product_id)

    if product:
        redis.hmset(cache_key, product)
        redis.expire(cache_key, 3600)
    else:
        # 缓存空值，防止穿透
        redis.setex(cache_key, 300, "NULL")

    return product
```

#### 3.3.3 缓存击穿处理

**解决方案**: 分布式锁

```python
def get_product_with_lock(product_id):
    cache_key = f"mall:product:info:{product_id}"
    lock_key = f"mall:lock:product:{product_id}"

    # 1. 查询缓存
    product = redis.hgetall(cache_key)
    if product:
        return product

    # 2. 获取分布式锁
    lock = redis.set(lock_key, 1, nx=True, ex=10)

    if lock:
        try:
            # 3. 再次查询缓存（double check）
            product = redis.hgetall(cache_key)
            if product:
                return product

            # 4. 查询数据库
            product = db.query("SELECT * FROM products WHERE id = %s", product_id)

            # 5. 写入缓存
            redis.hmset(cache_key, product)
            redis.expire(cache_key, 3600)

            return product
        finally:
            redis.delete(lock_key)
    else:
        # 6. 未获取到锁，稍后重试
        time.sleep(0.1)
        return get_product_with_lock(product_id)
```

#### 3.3.4 缓存雪崩处理

**解决方案**: 过期时间加随机值

```python
def set_cache_with_random_expire(key, value, base_expire):
    # 基础过期时间 + 随机值（0-300秒）
    random_expire = base_expire + random.randint(0, 300)
    redis.setex(key, random_expire, value)
```

### 3.4 库存扣减（防止超卖）

使用 Redis + Lua 脚本实现原子性操作：

```lua
-- 扣减库存 Lua 脚本
local stock_key = KEYS[1]
local quantity = tonumber(ARGV[1])

local current_stock = tonumber(redis.call('GET', stock_key) or 0)

if current_stock >= quantity then
    redis.call('DECRBY', stock_key, quantity)
    return 1
else
    return 0
end
```

**Python 调用示例**:
```python
def deduct_stock(sku_id, quantity):
    stock_key = f"mall:stock:sku:{sku_id}"

    # 执行 Lua 脚本
    result = redis.eval(lua_script, 1, stock_key, quantity)

    if result == 1:
        # 库存扣减成功，更新数据库
        db.execute(
            "UPDATE product_skus SET stock = stock - %s WHERE id = %s AND stock >= %s",
            quantity, sku_id, quantity
        )
        return True
    else:
        # 库存不足
        return False
```

## 4. 数据库索引策略

### 4.1 索引设计原则

1. **选择合适的字段建立索引**
   - WHERE 条件字段
   - ORDER BY 排序字段
   - JOIN 连接字段
   - 外键字段

2. **避免过度索引**
   - 索引会占用存储空间
   - 索引会降低写入性能
   - 频繁更新的字段谨慎建索引

3. **使用联合索引优化**
   - 遵循最左前缀原则
   - 区分度高的字段放前面

### 4.3 索引优化建议

#### 4.3.1 覆盖索引优化

**场景**: 查询订单列表

**优化前**:
```sql
-- 需要回表查询
SELECT id, order_no, total_amount, order_status
FROM orders
WHERE user_id = 123 AND order_status = 1
ORDER BY create_time DESC
LIMIT 10;
```

**优化后**:
```sql
-- 创建联合索引
CREATE INDEX idx_user_status_time ON orders(user_id, order_status, create_time, id, order_no, total_amount);

-- 查询时使用覆盖索引，无需回表
SELECT id, order_no, total_amount, order_status
FROM orders
WHERE user_id = 123 AND order_status = 1
ORDER BY create_time DESC
LIMIT 10;
```

#### 4.3.2 避免索引失效

**常见的索引失效场景**:
1. 使用函数处理索引字段
2. 隐式类型转换
3. LIKE 查询以 % 开头
4. OR 连接的条件
5. 不符合最左前缀原则

**示例**:

```sql
-- 错误：使用函数，索引失效
SELECT * FROM orders WHERE DATE(create_time) = '2026-03-04';

-- 正确：使用范围查询
SELECT * FROM orders WHERE create_time >= '2026-03-04 00:00:00' AND create_time <= '2026-03-04 23:59:59';

-- 错误：LIKE % 开头，索引失效
SELECT * FROM products WHERE name LIKE '%手机%';

-- 正确：使用全文索引或搜索引擎
-- 或改为后缀匹配
SELECT * FROM products WHERE name LIKE '苹果手机%';
```

#### 4.3.3 索引监控与维护

```sql
-- 查看索引使用情况
SELECT
    table_name,
    index_name,
    cardinality,
    seq_in_index,
    column_name
FROM information_schema.statistics

-- 查看未使用的索引
SELECT
    object_schema,
    object_name,
    index_name
FROM performance_schema.table_io_waits_summary_by_index_usage
WHERE index_name IS NOT NULL
AND count_star = 0
AND object_schema = 'mall'
ORDER BY object_schema, object_name;

-- 分析表
ANALYZE TABLE products;

-- 优化表
OPTIMIZE TABLE products;
OPTIMIZE TABLE orders;
```

### 6.2 常用查询示例

#### 6.2.1 用户相关查询

```sql
-- 查询用户信息
SELECT id, openid, nickname, avatar_url, phone, status, register_time
FROM users
WHERE id = 1;

-- 查询用户收货地址
SELECT id, receiver_name, receiver_phone, province, city, district, detail_address, is_default
FROM user_addresses
WHERE user_id = 1
ORDER BY is_default DESC, id DESC;

-- 查询用户订单列表
SELECT o.id, o.order_no, o.total_amount, o.order_status, o.pay_status, o.create_time
FROM orders o
WHERE o.user_id = 1
ORDER BY o.create_time DESC;

-- 查询用户订单详情
SELECT
    o.id, o.order_no, o.total_amount, o.pay_amount, o.order_status, o.pay_status,
    o.receiver_name, o.receiver_phone, o.receiver_address,
    oi.product_name, oi.sku_name, oi.price, oi.quantity, oi.total_amount
FROM orders o
LEFT JOIN order_items oi ON o.id = oi.order_id
WHERE o.id = 1;
```

#### 6.2.2 商品相关查询

```sql
-- 查询商品列表（带分页）
SELECT p.id, p.name, p.subtitle, ps.price, ps.stock, p.sales_count, pi.image_url
FROM products p
LEFT JOIN product_skus ps ON p.id = ps.product_id AND ps.status = 1
LEFT JOIN product_images pi ON p.id = pi.product_id AND pi.image_type = 1
WHERE p.status = 1
ORDER BY p.sort_order ASC, p.create_time DESC
LIMIT 0, 20;

-- 查询商品详情
SELECT
    p.id, p.name, p.subtitle, p.description, p.brand, p.sales_count,
    ps.id AS sku_id, ps.sku_name, ps.price, ps.stock, ps.sku_attributes,
    pi.image_url, pi.image_type
FROM products p
LEFT JOIN product_skus ps ON p.id = ps.product_id
LEFT JOIN product_images pi ON p.id = pi.product_id
WHERE p.id = 1 AND p.status = 1;

-- 查询分类商品
SELECT p.id, p.name, ps.price, p.sales_count, pi.image_url
FROM products p
LEFT JOIN product_skus ps ON p.id = ps.product_id AND ps.status = 1
LEFT JOIN product_images pi ON p.id = pi.product_id AND pi.image_type = 1
WHERE p.category_id = 1 AND p.status = 1
ORDER BY p.sort_order ASC, p.sales_count DESC
LIMIT 0, 20;

-- 查询热销商品
SELECT p.id, p.name, ps.price, p.sales_count, pi.image_url
FROM products p
LEFT JOIN product_skus ps ON p.id = ps.product_id AND ps.status = 1
LEFT JOIN product_images pi ON p.id = pi.product_id AND pi.image_type = 1
WHERE p.is_hot = 1 AND p.status = 1
ORDER BY p.sales_count DESC
LIMIT 10;

-- 搜索商品
SELECT p.id, p.name, ps.price, p.sales_count, pi.image_url
FROM products p
LEFT JOIN product_skus ps ON p.id = ps.product_id AND ps.status = 1
LEFT JOIN product_images pi ON p.id = pi.product_id AND pi.image_type = 1
WHERE p.status = 1 AND p.name LIKE '%手机%'
ORDER BY p.sales_count DESC
LIMIT 0, 20;
```

#### 6.2.3 购物车相关查询

```sql
-- 查询购物车
SELECT
    sc.id, sc.quantity, sc.is_selected,
    p.id AS product_id, p.name AS product_name,
    ps.id AS sku_id, ps.sku_name, ps.price, ps.stock,
    pi.image_url
FROM shopping_cart sc
LEFT JOIN product_skus ps ON sc.sku_id = ps.id
LEFT JOIN products p ON ps.product_id = p.id
LEFT JOIN product_images pi ON p.id = pi.product_id AND pi.image_type = 1
WHERE sc.user_id = 1 AND sc.is_selected = 1
ORDER BY sc.create_time DESC;

-- 计算购物车金额
SELECT SUM(ps.price * sc.quantity) AS total_amount
FROM shopping_cart sc
LEFT JOIN product_skus ps ON sc.sku_id = ps.id
WHERE sc.user_id = 1 AND sc.is_selected = 1;
```

#### 6.2.4 订单相关查询

```sql
-- 查询订单列表

WHERE o.user_id = 1

LIMIT 0, 20;

-- 按状态查询订单
SELECT o.id, o.order_no, o.total_amount, o.order_status, o.pay_status, o.create_time
FROM orders o
WHERE o.user_id = 1 AND o.order_status = 1
ORDER BY o.create_time DESC;

-- 查询订单详情
SELECT
    o.*,
    oi.product_name, oi.sku_name, oi.price, oi.quantity, oi.total_amount
FROM orders o
LEFT JOIN order_items oi ON o.id = oi.order_id
WHERE o.id = 1;

-- 查询订单状态日志
SELECT old_status, new_status, remark, operator, create_time
FROM order_status_log
WHERE order_id = 1
ORDER BY create_time ASC;
```

### E. 性能优化建议

1. **读写分离**
   - 主库负责写操作
   - 从库负责读操作
   - 使用中间件实现自动路由

2. **分库分表**
   - 当单表数据量超过 1000万 时考虑分表
   - 当单库数据量超过 1亿 时考虑分库
   - 按业务模块垂直分库
   - 按数据特征水平分表

3. **慢查询优化**
   - 开启慢查询日志
   - 定期分析慢查询
   - 优化慢查询SQL
   - 合理使用索引

4. **连接池配置**
   - 合理设置连接池大小
   - 避免连接泄漏
   - 监控连接使用情况

5. **监控告警**
   - 监控数据库QPS、TPS
   - 监控慢查询数量
   - 监控主从延迟
   - 监控磁盘使用率
