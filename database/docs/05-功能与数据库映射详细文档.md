### 1. 用户认证模块

#### 1.1 用户注册 🆕

**功能描述**：用户填写用户名、密码、手机号、邮箱进行注册

**涉及表**：
- ✅ **users** - 写入

**数据库操作**：
```sql
-- 插入新用户
INSERT INTO users (username, password, phone, email, nickname, register_time, status)
VALUES (?, ?, ?, ?, ?, NOW(), 1);

-- 参数说明
-- username: 用户名（必填，唯一）
-- password: 加密后的密码（必填）
-- phone: 手机号（可选，唯一）
-- email: 邮箱（可选，唯一）
-- nickname: 昵称（可选，默认为用户名）
```

**操作类型**：
- ✅ **INSERT** - 创建新用户
- ✅ **SELECT** - 检查用户名/手机号/邮箱是否已存在

**API端点**：`POST /api/users/register`

#### 3.3 商品列表（首页推荐）🏷️

**功能描述**：首页展示推荐商品（热销、新品等）

**涉及表**：
- ✅ **product_spus** - 读取
- ✅ **product_spu_tags** - 读取（标签筛选）
- ✅ **product_images** - 读取（主图）

**数据库操作**：
```sql
-- 基础查询
SELECT p.id, p.spu_code, p.title, p.subtitle, p.min_sale_price,
       p.max_line_price, p.total_stock, p.sold_num,
       i.image_url as primary_image
FROM product_spus p
LEFT JOIN product_images i ON i.spu_id = p.id AND i.image_type = 1
WHERE p.status = 1
ORDER BY p.sold_num DESC, p.id DESC
LIMIT 20;

-- 带标签筛选（如"热销"）
SELECT p.id, p.spu_code, p.title, p.subtitle, p.min_sale_price,
       p.max_line_price, p.total_stock, p.sold_num,
       i.image_url as primary_image
FROM product_spus p
LEFT JOIN product_images i ON i.spu_id = p.id AND i.image_type = 1
INNER JOIN product_spu_tags t ON t.spu_id = p.id
WHERE p.status = 1
AND t.tag_type = 'hot'  -- 或 'new'
ORDER BY p.sold_num DESC, p.id DESC
LIMIT 20;
```

**操作类型**：
- ✅ **SELECT** - 查询商品（带JOIN）

**API端点**：`GET /api/products/list`

#### 5.4 取消订单 ❌

**功能描述**：用户取消未发货的订单

**涉及表**：
- ✅ **orders** - 更新
- ✅ **order_status_logs** - 写入
- ✅ **product_skus** - 更新（恢复库存）

**数据库操作**：
```sql
-- 开启事务
BEGIN;

-- 1. 更新订单状态
UPDATE orders
SET order_status = 6,
    cancel_time = NOW(),
    cancel_reason = ?,
    updated_at = NOW()
WHERE id = ?
AND user_id = ?
AND order_status IN (1, 2);  -- 只有待付款、待发货可以取消

-- 2. 记录状态日志
INSERT INTO order_status_logs (order_id, order_no, from_status, to_status,
                               operator_type, remark, created_at)
VALUES (?, ?, ?, 6, 1, ?, NOW());

-- 3. 恢复库存
UPDATE product_skus
SET stock = stock + ?
WHERE id IN (
  SELECT sku_id FROM order_items WHERE order_id = ?
);

-- 提交事务
COMMIT;
```

## 数据库操作类型统计

### 按操作类型分类

| 操作类型 | 数量 | 说明 |
|---------|------|------|
| **SELECT** | 45+ | 查询数据（读多写少） |
| **INSERT** | 20+ | 插入新数据 |
| **UPDATE** | 15+ | 更新现有数据 |
| **DELETE** | 5+ | 删除数据 |
| **事务** | 3-5 | 复杂操作（创建订单、发货、审核） |

### 按频率分类

| 频率 | 操作 | 说明 |
|------|------|------|
| **高频** | 商品列表、商品详情、购物车列表 | 每次浏览都执行 |
| **中频** | 订单列表、用户信息、收货地址 | 用户主动查询 |
| **低频** | 用户注册、商品管理、售后审核 | 偶尔执行 |

### 按复杂度分类

| 复杂度 | 操作 | 说明 |
|--------|------|------|
| **简单** | 单表查询、单条插入 | 查询单个用户、添加地址 |
| **中等** | 多表JOIN、批量操作 | 商品列表、订单列表 |
| **复杂** | 事务、多步骤操作 | 创建订单、发货、售后审核 |

```sql
CREATE UNIQUE INDEX idx_cart_user_sku ON shopping_cart(user_id, sku_id);
```

### 2. 查询优化

**避免全表扫描**：
```sql
-- ❌ 不推荐
SELECT * FROM orders;

-- ✅ 推荐
SELECT * FROM orders WHERE user_id = ? LIMIT 20;
```

**使用字段选择**：
```sql
-- ❌ 不推荐
SELECT * FROM users WHERE id = ?;

-- ✅ 推荐
SELECT id, username, nickname FROM users WHERE id = ?;
```

### 3. 缓存策略

**适合缓存的数据**：
- 轮播图（5分钟）
- 商品分类（30分钟）
- 商品列表（1分钟）
- 配送费用（1小时）

**不适合缓存的数据**：
- 用户信息（实时变化）
- 购物车（实时变化）
- 订单状态（实时变化）

### 4. 分页限制

**所有列表接口必须分页**：
```sql
-- 默认每页20条
SELECT * FROM product_spus LIMIT 20 OFFSET 0;

-- 最多每页100条
SELECT * FROM product_spus LIMIT ? OFFSET ?;
```
