ENVIRONMENT CONFIGURATION
================================================================================

Required Variables:
  - NODE_ENV (production/development)
  - PORT (default: 3000)
  - DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD
  - JWT_SECRET (important for security!)
  - JWT_EXPIRES_IN
  - WECHAT_APPID, WECHAT_SECRET

Optional Variables:
  - UPLOAD_PATH (default: ./uploads/products)
  - MAX_FILE_SIZE (default: 2097152)
  - LOG_LEVEL (default: info)
  - LOG_PATH (default: ./logs)

================================================================================
DATABASE SETUP
================================================================================

Manual Setup:
  mysql -u root -p
  CREATE DATABASE wechat_shop CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  CREATE USER shopuser@localhost IDENTIFIED BY password;
  GRANT ALL PRIVILEGES ON wechat_shop.* TO shopuser@localhost;

Import Schema:
  mysql -u shopuser -p wechat_shop < database/schema.sql

================================================================================
NGINX CONFIGURATION
================================================================================

Location: /etc/nginx/sites-available/wechat-shop

Key Features:
  - HTTP to HTTPS redirect
  - SSL/TLS support (Let's Encrypt)
  - Reverse proxy to Node.js
  - Gzip compression
  - Security headers
  - File upload size limits

Enable Site:
  sudo ln -s /etc/nginx/sites-available/wechat-shop /etc/nginx/sites-enabled/
  sudo nginx -t
  sudo systemctl reload nginx

SSL Certificate:
  sudo apt install certbot python3-certbot-nginx -y
  sudo certbot --nginx -d api.yourdomain.com

================================================================================
MONITORING & MAINTENANCE
================================================================================

PM2 Commands:
  pm2 status              # Check status
  pm2 logs wechat-shop    # View logs
  pm2 monit               # Real-time monitoring
  pm2 restart wechat-shop # Restart service
  pm2 reload wechat-shop  # Zero-downtime reload

Docker Commands:
  docker-compose ps       # Check containers
  docker-compose logs -f  # View logs
  docker-compose restart  # Restart services
  docker-compose down     # Stop all

Database Backup:
  mysqldump -u shopuser -p wechat_shop | gzip > backup.sql.gz

Restore:
  gunzip < backup.sql.gz | mysql -u shopuser -p wechat_shop

================================================================================
SECURITY BEST PRACTICES
================================================================================

1. Always use strong, unique passwords
2. Never commit .env files to version control
3. Use HTTPS in production with valid SSL certificates
4. Keep dependencies updated regularly
5. Enable firewall and only expose necessary ports
6. Use environment variables for sensitive configuration
7. Implement rate limiting on API endpoints
8. Regular database backups with automated scripts
9. Monitor logs for suspicious activity
10. Keep system packages updated
