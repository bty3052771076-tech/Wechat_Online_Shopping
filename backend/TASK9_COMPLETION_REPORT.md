## API端点覆盖统计

### 用户端API (13个端点)
1. `POST /api/users/login` - 用户登录
2. `GET /api/users/profile` - 获取用户信息
3. `GET /api/products/list` - 获取商品列表
4. `GET /api/products/detail/:id` - 获取商品详情
5. `GET /api/products/categories/list` - 获取分类列表
6. `GET /api/products/categories/tree` - 获取分类树
7. `POST /api/cart/add` - 加入购物车
8. `GET /api/cart/list` - 获取购物车列表
9. `PUT /api/cart/update` - 修改购物车
10. `DELETE /api/cart/delete/:id` - 删除购物车商品
11. `POST /api/addresses` - 添加收货地址
12. `GET /api/addresses` - 获取地址列表
13. `PUT /api/addresses/:id/default` - 设置默认地址

### 订单API (3个端点)
14. `POST /api/orders/create` - 创建订单
15. `GET /api/orders/list` - 获取订单列表
16. `GET /api/orders/detail/:id` - 获取订单详情

### 管理端API (8个端点)
17. `POST /api/admin/login` - 管理员登录
18. `GET /api/admin/profile` - 获取管理员信息
19. `GET /api/admin/products` - 获取商品列表（管理端）
20. `POST /api/admin/products` - 添加商品
21. `PUT /api/admin/products/:id` - 更新商品
22. `GET /api/admin/orders` - 获取订单列表（管理端）
23. `PUT /api/admin/orders/:id/ship` - 订单发货

**总计**: 23个API端点测试

## 测试特性说明

### 1. 错误处理
- 每个测试用例独立使用try-catch捕获错误
- 测试失败不会中断整个测试流程
- 详细的错误信息输出
- 失败测试汇总在结果中展示

### 2. 数据验证
- API响应状态码验证
- 响应数据结构验证
- 关键字段存在性验证
- 数据类型和值验证

### 3. 认证流程
- 先执行用户和管理员登录
- 获取并存储JWT token
- 后续测试自动使用token
- 支持token有效性验证

### 4. 测试隔离
- 使用测试数据避免污染生产数据
- 添加的商品标记为测试商品
- 测试地址使用虚拟数据
- 订单创建使用测试商品

### 5. 结果统计
- 实时显示测试进度
- 统计通过和失败数量
- 计算通过率百分比
- 记录执行时间
- 根据结果设置退出码

## 退出码说明

- **退出码 0**: 所有测试通过
- **退出码 1**: 存在失败的测试

可以在CI/CD流水线中使用：
```bash

  echo "测试通过，继续部署"
else
  echo "测试失败，阻止部署"
  exit 1
fi
```
